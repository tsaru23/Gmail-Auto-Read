# Gmail 不要メール自動既読プログラム (GAS)

Gmailに届く「見る必要がないメール」を、あらかじめ指定したルールに基づいて自動的に既読にするGoogle Apps Script (GAS) プログラムです。

## 特徴とセキュリティ設計

- **セキュリティの考慮（機密情報の排除）**: 既読対象とするメールの「送信元アドレス」や「キーワード」などのフィルタ条件、およびスプレッドシートの接続情報をコード内に一切ハードコーディングしません。
- **外部スプレッドシート管理**: 既読にするルールはGoogleスプレッドシートで管理するため、コードを変更することなく動的にルールの追加・削除・変更が可能です。
- **直感的な日本語プルダウンUI**: 「送信元(From)」や「部分一致(Contains)」などの設定項目を日本語のプルダウンから選択できます。有効/無効の切り替えはチェックボックスで行えます。
- **AI不要で管理可能（使い方ヘルプ内蔵）**: システムを初期化すると、スプレッドシート内に「使い方ヘルプ」シートが自動生成されます。設定例が細かく載っているため、AIに相談しなくても自分自身で簡単にフィルタをカスタマイズできます。
- **スクリプトプロパティの利用**: スプレッドシートIDなどの環境固有情報は、GASの「スクリプトプロパティ」に保存します。これにより、本リポジトリのソースコードをGitHub等にパブリック公開しても、個人情報や特定の宛先情報が漏洩することはありません。
- **実行ログ機能**: どのルールにマッチして既読にしたかをスプレッドシートに自動記録します。

## 導入手順

### 1. 管理用Googleスプレッドシートの作成
1. 新しいGoogleスプレッドシートを作成します。
2. スプレッドシートのブラウザURLから、スプレッドシートID（`https://docs.google.com/spreadsheets/d/[この部分の文字列]/edit`）をコピーしてメモしておきます。

> ※ 初回プログラム実行時に、シート構成（入力制限プルダウン・チェックボックス付き）および「使い方ヘルプ」シートが自動で作成されます。

---

### 2. GASプロジェクトの作成とコードの配置
1. [Google Apps Script](https://script.google.com/) にアクセスし、「新しいプロジェクト」を作成します。
2. デフォルトの `コード.gs` に、本リポジトリの `main.js` のコードをすべて貼り付けます。
3. プロジェクトを保存します。

---

### 3. スクリプトプロパティの設定（セキュア接続）
1. GASエディタの左メニューにある **「プロジェクトの設定」**（歯車マーク）をクリックします。
2. ページ下部にある **「スクリプト プロパティを追加」** をクリックします。
3. 以下のプロパティを追加します。
   - **プロパティ**: `SPREADSHEET_ID`
   - **値**: [手順1でメモしたスプレッドシートID]
4. **「スクリプト プロパティを保存」** をクリックします。

---

### 4. 初回実行（シートの自動初期化）と権限の承認
1. GASエディタの「エディタ」画面（`<>` マーク）に戻ります。
2. 上部の実行関数リストから `autoMarkAsRead` を選択し、**「実行」** ボタンを押します。
3. 初回実行時、Gmailおよびスプレッドシートへのアクセス権限の承認を求められます。画面の指示に従って承認を行ってください。
4. 実行後、スプレッドシートに **`FilterRules`** （ルール設定シート）、**`使い方ヘルプ`**、および **`Logs`** （実行ログシート）が自動的に生成されます。

---

### 5. 定期実行トリガーの設定
1. GASエディタの左メニューにある **「トリガー」**（時計マーク）をクリックします。
2. 右下の **「トリガーを追加」** をクリックします。
3. 以下のように設定して保存します。
   - **実行する関数を選択**: `autoMarkAsRead`
   - **デプロイ時に実行する関数を選択**: `Head`
   - **イベントのソースを選択**: `時間主導型`
   - **時間ベースのトリガーのタイプを選択**: `分ベースのタイマー`（または時間ベース）
   - **時間の間隔を選択（分）**: `10分おき`（任意の頻度）

これで、設定した間隔で自動的にGmailの不要メールが既読になります。

---

### 6. ルール設定シート (`FilterRules`) の使い方

スプレッドシートの `FilterRules` シートを開くと、以下のカラムが用意されています。

| A列 | B列 | C列 | D列 | E列 |
| :--- | :--- | :--- | :--- | :--- |
| **種別 (Type)** | **キーワード (Pattern)** | **一致方法 (MatchType)** | **説明 (Description)** | **有効にする (Active)** |
| プルダウンから選択<br>(`送信元(From)`, `件名(Subject)`, `本文(Body)`) | 直接手入力 | プルダウンから選択<br>(`完全一致(Exact)`, `部分一致(Contains)`, `正規表現(RegExp)`) | 直接手入力（ルールの説明・メモ書き用） | チェックボックスをオン/オフ |
| (設定例)<br>`送信元(From)` | (設定例)<br>`newsletter@example.com` | (設定例)<br>`完全一致(Exact)` | (設定例)<br>`特定のメルマガを完全に既読化` | (設定例)<br>☑ (チェックONで有効) |

具体的な活用方法や他のルール設定例（重要な連絡は残しつつ不要なイベントメールだけを消す方法など）は、スプレッドシートに自動作成される **`使い方ヘルプ`** シートに詳しく解説されています。AIを使用しなくても、そのシートの例を見ながらいつでもルールを自由に追加・編集できます。

---

### 7. メールをエクスポートしてAIと相談しながらフィルタを決める（オプション）
どのようなメールが届いているかをAI（Antigravity）と一緒に確認し、自動既読化ルールを安全に策定するための機能です。

1. GASエディタの上部関数リストから `exportEmailsForAnalysis` を選択し、**「実行」** ボタンを押します。
2. 実行後、スプレッドシートに `TempEmails` シートが作成され、直近50件の未読メール情報（送信元、件名、本文の一部）が書き出されます。
3. このシートの内容（個人を特定しうる詳細な本文情報は除いた上で、送信元や件名のパターンなど）をチャットに共有いただくことで、AIが最適なフィルタルールを推測・提案します。
4. ルールが決定した後は、スプレッドシートの `TempEmails` シートを手動で削除（またはデータを消去）することで、スプレッドシートをきれいな状態に保てます。

本スクリプトの使用によって、重要なメールが誤って既読になり見落とされた場合など、いかなるトラブルや損失に対しても製作者は責任を負いません。フィルタの設定（特に `正規表現(RegExp)` や `部分一致(Contains)`）は慎重に行い、動作検証を十分に行ってから運用を開始してください。
